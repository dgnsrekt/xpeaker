🔊 Xpeaker

Reads X / Twitter posts aloud through chrome.tts + Supertonic voices — no host server.

Per-author voices (override auto)

Word highlighting

Highlights the current word as it's read. Needs the voice engine to emit word events; if it doesn't, the caption still shows the full text.

On-device AI (transformers.js — no server)

The model downloads once (from Hugging Face) and is cached. First run is slow; after that it's offline. WebGPU is much faster but isn't available everywhere — Xpeaker falls back to WASM. Summary mode (bar) digests a thread, then reads the digest aloud.

Keyboard shortcuts

Alt+R read under cursor · Alt+T thread from there · Alt+S stop · Alt+N/Alt+B next/back · Alt+Space pause · Alt+↑/Alt+↓ speed

Changes save automatically.
